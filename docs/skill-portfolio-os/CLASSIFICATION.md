# Classification: Free · Open-Core · Gated · Private · Product

## Tier definitions

| Tier | License / access | Intent | Examples |
|------|------------------|--------|----------|
| **T0 — Runtime-only** | Local Hermes only | Host/agent glue never meant as a product | `windows-phone-link-search-safety`, profile cruft, path quirks |
| **T1 — Free community (MIT)** | Public GitHub, Skills Hub, awesome lists | Portable process excellence; mindshare; SEO; inbound | Generic CLI orchestration patterns, TDD, debugging, skill authoring, sanitized Model Council, workflow-tier-plugin |
| **T2 — Open-core free surface** | Public repo + “premium layer” pointer | Flagship OS demos; free forever core | ACOS public skills, SIS public substrate docs, awesome-* lists, peak-performance CLI free tier |
| **T3 — Gated / premium** | Paid vault, cohort, license key, private repo | Production systems, brand packs, advanced loops | `realityarchitect-vault`, full Brand Image System packs, multi-brand CoE runbooks, personal Velora instance templates with real data seams |
| **T4 — Private sovereign** | Private only (FrankX, gencreator.ai, profiles) | Strategy, legal, tax, personal, unreleased | Empire plans, grant drafts, finance OS with accounts, Arcanea unreleased canon ops |
| **T5 — Productized** | SKU, pricing, support, distribution | Revenue line | GenCreator CoE kits, Velora product, Arcanea app, MCP servers, Academy, Guardian Factory service |

### Open-core rule (default business model)

```
FREE (T1/T2)          GATED (T3)              PRODUCT (T5)
portable skills  +  production kits     =  SKUs & services
awesome lists       brand packs
CLI patterns        personal OS templates
eval harnesses      multi-brand ops
```

**Do not** make *all* free (kills moat + confuses buyers).  
**Do not** make *all* paid (kills stars, SEO, talent, ecosystem gravity).

## Classification tests (run before publish)

### Promote to FREE (T1) only if ALL true

- [ ] No API keys, tokens, emails, phones, health/finance PII  
- [ ] No hard-coded `C:\Users\frank`, OnePlus, Phone Link as mandatory  
- [ ] No unreleased product strategy or pricing secrets  
- [ ] Works on macOS/Linux/Windows with documented prerequisites  
- [ ] Description starts with trigger (“Use when…”)  
- [ ] Verification checklist present  
- [ ] Related skills resolve publicly (or optional)  
- [ ] Genericized examples (no private repo names required)  

### Keep PRIVATE / GATED if ANY true

- [ ] Host-specific safety that is personal-only (Frank machine ban details → **extract generic Windows recursion advice to free; keep Phone Link incident log private**)  
- [ ] Multi-brand ICP, revenue, legal entity detail  
- [ ] Personal training logs, N-of-1 physiology, banking  
- [ ] Full brand visual DNA / Soul IDs / proprietary assets  
- [ ] Grant applications, client work, partner NDAs  
- [ ] Incomplete stubs that would shame the brand if public  

### PRODUCT if

- [ ] Named buyer + recurring value  
- [ ] Install/onboard path  
- [ ] Support / update story  
- [ ] Clear free vs paid boundary

## Anti-patterns

| Bad | Why |
|-----|-----|
| Dumping entire `~/.hermes/skills` public | Secrets, sediment, stubs, brand damage |
| Gating basic TDD/debug skills | Kills trust; those should free |
| Freeing full multi-brand CoE ops | Gifts the moat |
| Dual public copies that drift | Use one public SoT + tags |
| “All free forever” with no product | No scale path for ops cost |
| “All paid” skill wall | No gravity, no talent, no SEO |
