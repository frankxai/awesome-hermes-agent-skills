# Product Map — What This Skill Portfolio Becomes

## Strategy thesis

Skills alone are **not** the business. Skills are:

1. **Runtime leverage** (you ship faster)  
2. **Open-core gravity** (community + SEO + talent)  
3. **Product ingredients** (kits, OS, MCP, academy, services)

Revenue comes from **packaged outcomes**, not from hiding basic debugging skills.

```
Free skills & awesome lists  →  trust + distribution
Open-core OS repos           →  adoption + freemium funnel
Premium kits / vaults        →  high-margin digital products
SaaS/apps (Arcanea, Velora)  →  recurring
CoE / install services       →  high-ticket human+agent delivery
```

## Pricing posture (guidance, not legal)

| Tier | Typical model |
|------|----------------|
| T1 Free | MIT, stars, attribution, inbound |
| T2 Open-core | Free core forever; paid packs/hosting |
| T3 Gated | One-time kit or subscription vault |
| T5 Product | SaaS / app / academy / service |

**Do not gate:** basic process excellence, install docs, safety patterns (generic).  
**Do gate:** full brand systems, personal OS instances, production CoE graphs, proprietary media kits.

## What is NOT a product (keep free or internal)

- Generic plan/TDD/debug  
- Sanitized CLI discovery  
- Skill authoring standards  
- Host-only Phone Link skill (internal; generic Windows note free)  
- Incomplete stubs
